Editorial

01.

Philosophy

Design Logic

Editorial Machine draws from the confidence of mid-century advertising and the discipline of newspaper design. Headlines command. Body text serves. Rules organize.

This system treats typography as the primary interface element. Display type does the heavy lifting—announcing, persuading, organizing. The supporting elements (color, space, imagery) exist to amplify typographic hierarchy, never to compete with it.

Every component earns its place through function. Decoration without purpose is prohibited.

Optimizations

  • Headline confidence — Display type carries the message; size and weight signal importance
  • Executive scanning — Information hierarchy supports quick comprehension
  • Editorial rhythm — Consistent vertical spacing creates predictable reading patterns
  • Rule-based structure — Horizontal rules replace boxes and backgrounds for organization
  • Warm authority — Paper tones add warmth without sacrificing professionalism

Constraints

  • No rounded corners. Rectangular forms only.
  • No gradients or shadows beyond functional offset.
  • Color is structural (rules, ink, paper). Never decorative.
  • Photography is documentary. No lifestyle imagery.
  • Animation is state-change only. No aesthetic motion.

03.

Typography

Type Families

Display: Fraunces — A variable serif with optical sizing, high stroke contrast, and soft, rounded details. The "wonky" axis adds subtle personality without compromising legibility. Use at large sizes where its character becomes visible.

Interface: Inter — A neutral sans-serif optimized for screens. Handles body text, labels, and UI elements. Its job is to stay invisible.

Body / Inter 400 / Paragraph

Body text uses Inter at 16px with generous line height (1.75). The measure is constrained to 38rem for optimal readability. Paragraphs are separated by the line height value.

Small / Inter 500 / Labels

Form labels, table headers, and secondary interface text

Caption / Inter 600 / Metadata

Section Labels • Timestamps • Categories

Typography Rules

  • Measure: Body text maximum 38rem (≈65 characters)
  • Line height: 1.75 for body, 1.0–1.2 for display
  • Display weight: 700–900 for headlines. Never use display type at body weight.
  • Italic use: Reserved for subtitles, pull quotes, and editorial voice. Never for emphasis.
  • ALL CAPS: Labels and metadata only. Requires letter-spacing ≥0.1em.
  • No underlines except links. Use weight for text emphasis.

12-column grid with responsive column counts. Gutters maintain 1.5rem. Page maximum: 72rem. Content areas use asymmetric layouts when editorial hierarchy benefits.

05.

Iconography

Icon Style

Icons are functional glyphs, not decorative elements. Stroke-based, 1.5px weight at 24px, squared corners. Icons inherit ink color and never introduce new colors.

Property Specification
StyleOutlined, stroke-based
Stroke weight1.5px at 24px
CornersSquare (no radius)
Grid24×24px
Sizes16px, 20px, 24px

Icon Rules

  • Icons accompany text labels. Icon-only requires tooltip + aria-label.
  • Icons inherit current text color. No separate icon colors.
  • Minimum touch target: 44×44px regardless of icon size.
  • If the icon doesn't add clarity, remove it.
06.

Imagery & Media

Image Role

Imagery is documentary. It shows the product, demonstrates the interface, or visualizes data. Stock photography and lifestyle imagery are prohibited. If text can communicate the message, imagery is unnecessary.

Image Treatment

  • Aspect ratios: 16:9 (hero), 4:3 (card), 3:2 (editorial), 1:1 (thumbnail)
  • Border: 2px solid Ink for all images
  • Background: Paper Aged or Paper Stock for product shots
  • No filters, overlays, or color treatments
  • Screenshots include browser chrome or device frame

Actions

Button Rules

  • One primary button per view
  • Button text is verb-first: "Save changes" not "Changes"
  • Minimum width: 120px. No full-width buttons unless in narrow containers.
  • Border: 2px (standard), 3px (large)
  • No rounded corners. Square edges only.
10.

Feedback

Alert Examples

Changes saved successfully.
Unable to save. Check your connection and try again.
Your session will expire in 5 minutes.

Feedback Rules

  • Messages are specific: "Document saved" not "Success"
  • Errors don't blame: "Enter a valid email" not "You entered an invalid email"
  • Left border (6px) signals severity
  • Success auto-dismisses after 5s. Errors persist until resolved.
